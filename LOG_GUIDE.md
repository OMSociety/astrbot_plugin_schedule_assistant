# ═══════════════════════════════════════════════════════════════
# 日志级别规范
# ═══════════════════════════════════════════════════════════════
#
# DEBUG   - 每次调用、详细数据、失败细节（频繁但不关键）
# INFO    - 定时任务启动/完成、配置加载、关键流程里程碑
# WARNING - 配置缺失、可恢复错误、需要关注但不影响功能
# ERROR   - 功能不可用、异常退出
#
# 原则：用户操作触发的用 INFO，后台静默的用 DEBUG，避免刷屏
